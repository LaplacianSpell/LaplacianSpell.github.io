数据类型

如何理解数据类型

所有的数据都是按照一定的编码方式编码成的 bits，按照这个逻辑，C 中的数据类型实际上由两个内涵：

  • 数据所占有的内存大小
  • 如何解码对应内存大小上的 bits，获得相应的数据实际意义。

C++ 内支持的数据类型

  1. 基本类型（算数类型）

    • 整型（int, 4 bytes on 64bits）
    • 浮点型 (float 单精度浮点型，double 双精度浮点型 4 or 8 bytes)
    • 字符型 (char, 1 bytes on 64bits)

  2. void 类型

  3. 派生类型

    • 数组，结构体
    • 枚举类型

接下来的是如何在 C 中使用变量

声明与定义变量

  1. 声明变量是向编译器保证存在某个变量是某某变量名，并且对应的变量名拥有给定的数据类型。以让编译器在不清楚变量的细节的情况下还能继续编译。

告诉编译器，该文件中的变量 a 为一个外部变量，在其他的文件或者同一文件之后的某个代码块定义（如果不加，会报错，因为当前文件中会出现未经声明就使用的变量名）.变量可以声明多次，但是只能在一个文件的一个地方定义一次

  1. 定义变量是给变量分配空间和地址，又分为初始化和不初始化的定义。
  1. 函数的声明和定义：声明 int func()，定义 int func() { /.../ }

作用域是变量的作用范围，一般来说，变量声明的位置以及存储类会影响变量的作用范围，

变量存储类

存储类表明了一个变量他的作用范围，生命周期

  • auto 默认存储类，只能修饰局部变量
  • register 定义存储在寄存器中，可以快速访问
  • static 指定变量可以在整个程序的生命周期中不被销毁

内存

关于C++编译的问题可以看 CSAPP 第一章的读书笔记

当我们开始执行某一个程序的时候，操作系统在一开始就会分配给程序一些内存。

这部分内存分成两类：

  1. static 内存区域
    这部分从程序开始到结束，一直占用
    • 全局变量
    • 程序代码
  2. 栈内存
    • 局部变量
    • 函数参数
    • 返回值

其中栈内存是以函数调用栈的形式存在的，函数调用栈是操作系统赋予程序的一段用来存储调用函数所产生的内存开销的内存，而程序的每一次函数调用就会在调用栈里以栈帧为单位分配一段大小合适内存（由程序自行分配），函数返回后，栈帧内存会释放

因此，当递归深度过大时，栈帧没有得到释放就会占满调用栈内存，造成Stack overflow

此外，程序可以向操作系统申请的程序保存在堆上

堆和栈等等都是操作系统对进程进行内存抽象的产物 (关于虚拟内存的知识可以看 CSAPP).

函数重载，函数重写和函数覆盖

  • 函数重载指的是在同一作用域内有多个名称相同但参数列表不同，进而完成不同功能的函数

内联函数

在定义一个函数的时候，我们可以在之前添加关键字 inline 将函数变为内联函数，作用类似于宏定义，适用于反复使用的短小函数。

下面的两个是在类的拷贝构造函数中的性能提升

引子

一个类的单参数构造函数调用存在十分隐晦的情况，如果这个参数巨大，那么一旦调用构造函数则会造成不菲的开销

一个类通常存在拷贝构造函数，常常在类的对象作为参数传入函数中和作为函数的返回值的时候悄无声息的被调用，此时如果这个类无比巨大，那么将会带来极大的开销（夸张了一点，反正就是会带来一些开销）

explicit

在单参数构造函数前加上这一个关键字，防止隐式调用

右值引用

左值和右值，在等号左边的值，可以取地址的值为左值，在等号右边的值，不能取地址的值为右值

一般比较常见的右值有表达式 (a+b…)，函数返回值（除了返回左值引用）…

右值引用的意义在于延长右值的生命周期，一般右值在完成赋值操作之后即刻销毁。

这里很重要的一点是，我们借由右值引用，没有产生在拷贝构造函数 A(A& a) 中需要进行深拷贝复制原有数据的开销，我们只是把引用中的数据通过指针交给了新的对象。不需要额外开拓空间进行复制。

多文件编译

为什么有时候在写多文件 C++ 的时候会出现调换一下头文件就能编译通过？

因为在多文件的链接过程，文件的顺序十分重要

编译器会对文件从左向右扫描，如果遇到一个文件中定义了一个符号，编译器会尝试将这个符号链接到前面的文件中（把前面所有的声明都接上这个定义），此时如果一个符号的定义出现在声明文件之前，那么这个声明就不会被定义，会报 LNK:ERROR .

跨平台和跨语言也会导致链接错误，因为链接器的会导致不同平台不同语言对应的一些名称不一样，从而不能完成链接
