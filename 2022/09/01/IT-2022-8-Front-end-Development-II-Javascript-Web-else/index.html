前端开发的一些名词，引子

JavaScript

当我们的网页拥有了内容以及内容的排版之后仍然是不够的，我们还需要实现用户与网页之间的交互，js 是最常用来完成用户与网页交互关系的脚本，是一门解释性（or we say it 即时编译语言）.用来控制网页上各种信息的流动。

js 的标准是由国际组织 ECMA 制定的，但是标准中只约束的语法，没有对实现进行约束（比如同属之下的还有 JScripts 语言），且不同的浏览器中会对运行代码有不同的实现，比如 Chrome 浏览器中使用的引擎就是 V8.

Node.js

将 V8 与一些包和环境支持进行打包就变成了可以脱离浏览器运行 js 的环境了，这个环境就是 Node.js, 脱离浏览器的 Js 运行时

类比一下 python，我们创造一个虚拟环境 enve，每一个虚拟环境拥有自己的解释器和对应的包（numpy, matplotlib，pandas…）其自带包管理器是 pip…

Node.js 也有自带的包管理器 npm，但是我们常用第三方包管理器 yarn, 我们利用包管理器将相应的包从服务器拉取到本地的开发环境中。

前端框架

前端框架有 React，Vue.js，Bootstrap 等等，就是封装的好一点，让你干很多事变得方便一点，把 HTML, CSS, Javascripts 这些玩意包装好（最后还是会解释为上述语言来实现它）.

TypeScript

ts 是 js 的一个超集，由于 js 是一个弱类型的语言（数据类型没有强制固定，也就是说你可以将一个 int 类型赋值给 char 而不报错），为了避免产生一些奇怪的 bug，就产生了 Ts，Ts 拥有类型检查的功能

依赖 Node.js 的 Javascripts 运行时配置。

之后可以在环境中找到 Node.js 与对应包管理器 npm，下面通过 npm 下载 yarn

JavaScriptical 语法糖

下面总结一些语法糖

A.

JavaScriptical(不是)：变量转化为字符串

因为重载的运算符会将任何加在字符串上的变量变为字符串

B. ...运算符

可以展开对象键值对，常用来建造新对象

这里 ...obj 的含义就是将 obj 对象展开成键值对，也就是说这里 another 对象就获得了 a: 1、b: 2、c: 3 这三个键值对。而如果后续的键值对和 obj 之中的发生重合（如这里 another 中重新定义了 c 这个属性），则后者覆盖前者。所以这里新定义的 c: 2 覆盖掉 obj 之中的 c: 3。

这个语法的重要用途就是，如果我需要根据一个已有对象 obj 构造一个新对象 another，新对象和已有对象仅有一个键值对不同，就可以用这个语法。
