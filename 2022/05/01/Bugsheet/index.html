网站诞生的血泪史--一些使用Hexo魔改过程中部分遇到的bug与网站的未来方向

Bug Sheet

一.Syntax Error:Unexpected or Invalid token ‘…’

1.1 Description

这个Bug是因为自己在新建page的时因为替换不同的主题会产生不满足新得主题得不同的page页内容，比如：

在Butterfly theme:

在Metery theme

在我从Metery主题替换到Butterfly主题时候，friends页的名称以及其他页面的index文件内容也需要进行相应的调整。

2.Method

在source中删去Archive文档。（这一页不需要新建相关的文档）
注意之后一定要仔细阅读相应主题的 Readme 说明文件，按照给出的设定来调整 Page-directory name还有 index 文件，要不然就会想当然的新建一些无法解析的东西

二.图床不显示

2.1. Description

在我修改所有的设置之后，如果我在网站文件根目录Git bash中只是执行以下命令：

之后在端口local:4000查看网页的话，不会显示图床，只有Deploy到git仓库之后渲染才会出现。（这个不知道是不是Bug）

还有的是即使Deploy之后也无法显示的图片。

2.2 Method

解决方案是注意YML中书写的缩进格式(比如在冒号之后的空格之类的)，否则相关的图像可能会渲染不出来。
js还有css，html等Web开发的知识还是太少了，希望暑假好好学学。至少搞明白Hexo的一些运行机理.

三.Deploy之后不显示设置的改变

3.1. Description

Deploy之后刷新网页仍然不行。

3.2. Method

需要手动清理网页缓存，之后再加载，一般没问题了

四.TypeError: ‘…’/flink.pug

4.1. Description

这个Bug是因为配置Link界面内容的时候YML文件设置格式有问题导致网页无法解析。

4.2 Method

只需要按照Readme中保留每一个内容框架即可,我是把link.yml的友链部分的内容全部注释掉了（没有留下任何东西），事实证明还是需要留下至少一个的。
