什么是 Git

Git 是一个分布式版本控制系统，是一种命令行程序

分布式就是每个人都可以创建独立仓库管理，在本地进行操作。

对于某个大项目，可能有许多的开发者，如何统筹和协调不同开发者的不同的进度？我如何查看每一次改动的版本？我们使用版本控制系统，最常用的就是 Git.

Git 大致通过以下方式统筹协调：（伪算法）

  • 开发者从资源库（也就是所要开发的内容存放的地方，可能是一个远程服务器上的某个文件夹等等）克隆一个一模一样库到自己的开发平台（计算机）上。
  • 我们可以对自己平台上的这个克隆库进行文件的添加，删除，修改。如果在开发过程中库有更新，我们还能及时的更新。
  • 在提交我们的修改之前，可以进行检查。
  • 检查完毕之后可以提交我们的修改到资源库中，供其他开发者克隆，更新。
  • 如果发现错误，可以撤回提交，修改后再次提交。

当配置好 Git 环境之后，可以设置使用者信息（姓名和邮箱）.

之后我们引入基本的概念：工作区，暂存区和版本库，对象库等

工作区

工作区就是你的 Git 仓库内部的目录树，通俗来说，就是你仓库里面文件以及他们组成的树状结构.是我们存放开发文件的地方

版本库

当我们设置了 Git 仓库之后，会在我们当前所处的文件夹内生成一个 /.git 文件，这个文件不属于工作区，是用来存放工作区文件变动的相关信息的文件夹，即版本库，也是 Git 相关指令所操作的地方

暂存区，对象库

对象库是 /.git/objects 这是一个文件夹，

暂存区是 /.git/index ，这是一个文件，起到类似于字典中索引（目录）的作用，里面内容指向的对象库中的对象。

如果想回退版本，使用 reset

每一次 commit 就是将缓存中的目录写到 master 中。因此 commit 记录的是你提交之后与上一次提交的改变。

三种回退

–mixed 参数是 Default，用于重置暂存区的文件与上一次的提交 (commit) 保持一致，工作区文件内容保持不变。会删除 log 中的 commit

常用：–hard 直接删除之前的 commit 并恢复所有区的样子到指定的 commit（注意，都是该 commit 后的状态）

HEAD0/HEAD 表示当前版本
HEAD1/HEAD^ 上一个版本
HEAD2/HEAD^^ 上上一个版本
HEAD3/类推 上上上一个版本

缺省+HEAD 可以用来清空缓存

总结操作

操作的核心要点在于理解 add，实际上 add 指令可以更好的描述为同步缓存中某个文件与工作区中该文件的状态，因此修改完必 add，是否失去同步可以查看 status.其他的一些操作也是围绕着缓存区与 commit 来辨认的。

gitignore

可以简单理解为文件黑名单，防止 Git 仓库太大，避免提交一些对于复现项目多余的文件。

只要在.gitignore文件里直接添加文件名就好。

在 github 上对对应工具有自身的.gitignore 模板，使用相应的工具记得添加。

Github

创建 ssh 并粘贴到 Github 上是第一步，这样就能让当前终端与 Git 服务器交互

为了开发团队的维护方便，要么被开发团队添加到 Collaborator，要么在别人的 github 仓库那里 fork 到我的 github 仓库那里，之后在 fork 到我 github 下的仓库只和我的仓库进行 remote 操作。如果我想把我修改的东西合并到原来的仓库，就去提交一个 PULL REQUEST(P.R.),直接在自己的仓库 PR 即可，之后等待开发团队接收或者拒绝 merge 我的 pr.
