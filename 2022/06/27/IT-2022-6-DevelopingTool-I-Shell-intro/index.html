文件

文件是以计算机硬盘为载体所存储的信息集合.硬盘大致分为机械硬盘 (HDD,Hard Disk Drive),以及固态硬盘 (SSD，Solid State Drive)，而文件就是机械硬盘上面磁道上许多不同极化的集合，或者是固态硬盘上面的一系列不同电平的集合。极化形式和电平只采取 0，1 分类

在硬盘里使用 0 和 1 来记录信息。而将我们所熟悉的信息 (文本信息，声音信息，图像信息等) 转化为能在硬盘上存储的信息，则需要将原有的信息进行编码，将信息转化为二进制 bits 序列。同样的，我们读取的时候也顺着编码方式进行解码，从而将信息还原出来。

在计算机中文件通常有文件名，其中“.”后面称为后缀名，标注的是文件的编码方式，据此简单分类

纯文本文件

对纯文字进行编码的文件，比如.txt, .cpp, .html 等等。

对文本常用的编码方式有 ASCII，UTF-8，GBK(很蛋疼的一件事就是一些汉语文本按照国标编译和 UTF-8 编译的情况是不同的…因此会出现乱码)

程序文件

高级语言 (C，C++，python 等) 写出来的文件.cpp, .py 等本质上只是一些文本。

对于编译型语言需要经过相应的编译器/解释器进行编译，转化为一条一条的机器指令序列，这些指令通常是一串 64bits/32bits 流，用来告诉 CPU 该干什么：调用哪个寄存器，写入什么数据等等。形成可执行文件

这里涉及到指令集等等计算机指令架构的知识，也就是通用计算机的思想：利用有限完备指令集，即一些有限的，通用的操作，将大任务转化为这些操作的有序序列，只需依次进行这些基本操作，就可以完成任务

经过编译链接过后产生的.exe 文件 (即机器指令序列) 就是程序文件的一种。

媒体文件

对声音，图像等进行编码的文件以及他们的组合，比如声音文件.mp3, .wav 等，图像.jpg， .png 等，还有图像 + 声音的视频文件，图像＋文字＋视频＋声音的.pptx, .docx 等等，为了减少空间占用的压缩文件.rar .zip 等。

打开媒体文件需要相应的程序文件对他进行解析

操作系统 (OS, Operating System)

我们平时使用的例如 QQ，微信，浏览器 Chrome，游戏平台 Steam，阅读器 WPS，科学计算软件 Mathematica 等等程序，都是在系统程序基础上实现更高层次功能的程序。

而系统程序则上接高层次软件，下链接硬件。

他承担了对于硬件的调度，协调不同软件同时运行对于一些资源的竞争（并发），提供高层次软件公用的基础工具的程序 (比如编译器，数据库管理，存储器格式化，文件系统管理，用户身份验证，驱动管理，网络连接等) 等等更加基础和公共的功能。

可以将系统程序粗略分为

  • 系统应用：编译器，文件系统管理 (即提供基本的新建文件，重命名文件，读取文件等等功能的而程序)，网络链接（即按照协议接入互联网实现信息交互等功能的程序）等等，系统应用可以简单理解为计算机运行的基础工具

  • 操作系统
    操作系统主要为了实现以下功能：

    • 处理器管理：进程控制，进程同步，进程通信，作业进程调度
    • 存储器管理：内存分配，内存保护，地址映射，内存扩充
    • 设备管理：缓冲管理，设备分配，设备处理
    • 文件管理：存储管理，目录管理，读写管理和保护
    • 提供接口：系统调用，命令接口和 GUI

这些名词其实我很多都不明白，写在这里仅仅提供一个对 OS 的宏观认识，可以体会操作系统与系统基础应用软件等等其他概念之间的区别。

Linux 系统的简单介绍

一个操作系统大致分为内核和系统软件，我现在使用的是 Ubuntu，是 Linux 的一个发行版，即将内核与相应的基础软件打包。

Linux 的哲学：一切皆文件

Linux 的文件系统管理

在文件管理系统中，文件之间通常具有一定的层次关系，呈现出树状结构 (Hierarchy)

类 Unix 系统与 Windows 在文件系统管理上的小区别

在类 Unix 系统 (Linux, MacOS 等) 中文件路径以根目录开始，也就是’/‘,再也不能回退的层级; 而 Windows 系统则是以盘符来区分。也就是’C:',’D:'等等

有什么不同呢？（下面的阐述不保证严谨，经验主义，完全按照我个人的理解）
这里涉及到一些关于盘符分配以及挂载点等知识

  • Windows 系统中 C，D 这些盘并不一定是不同的硬盘，而是对同一个硬盘不同的分区，每一个分区有自己独立的树状结构，而盘符则可以简单的理解为对这些分区的命名，也就是该分区类似于“标题目录”的东西。每个分区有他自己的根节点。（在 Terminal Shell 中有跳转盘符的命令）

  • 在类 Unix 操作系统中，所有的文件和目录都被组织成以一个根节点开始的倒置的树状结构。而类 Unix 系统实行的是将某一个分区的内存挂载到相应的目录上，相应的目录就是挂载点

基于 Linux 的命令行程序简单介绍，概念辨析

Terminal/Shell/Bash/SSH…这些之间的关系是什么

  • Shell 是一个程序，用来输入命令与操作系统内核交互，封装保护操作系统的内核，是覆盖在 core 与用户之间的 shell
    • 1.（命令行形式）解析你输入的字符串，寻找字符串对应的程序（如果是绝对路径则按照路径寻找，如果是单词则前往环境变量中寻找）
    • 2.（图形形式）比如任务管理器，文档管理器。
    • 3.环境变量就是一系列放命令行程序的文件夹，如果我直接输入的字符串是这些文件夹中某一个文件夹里的一个程序，则可直接执行，因此我们称之为“命令行程序”，简称命令

下面给出小栗子
例子：寻找并执行 date 程序

也可以为命令行程序添加输入

程序与输入之间有一个分隔符，区分程序和参数；这一点 Shell 和图形界面没有什么不一样的地方。同时，程序还提供标签（类似于重载），指示程序执行不同的行为

下面简单的阐述二者的相同点，Windows 系统下 CPU 与 GPU 会根据我放在 C 盘下用户目录的 Desktop 文件中内容渲染桌面 GUI（快捷方式等等），而我点击相应的快捷方式来指挥操作系统帮我寻找并运行程序，等价于我在 Shell 中利用 Shell 输入相应的字符串，Shell 帮我解析字符串并寻找相应的程序，并告诉操作系统内核执行程序。

Shell 命令，Shell 脚本与 SSH

Shell 命令有以下几种类型

  • shell 函数（见之后的 shell 脚本内容）
  • 别名 alias（为复杂的命令命属于自己的名称）
  • 脚本（多条命令按行组合）

bash 脚本的哲学：一切皆字符串，你在写的一切东西都是字符串，命令也是字符串，只不过解释器帮你解析了而已

shebang 与 脚本传参

在脚本首行撰写 shebang

当第一行被执行的时候，内核会寻找该路径之下的程序作为该脚本的解释器

如果没撰写 shebang，那么我们当前 shell 就是执行脚本的 default 解释器.(指按照以下方式执行的情况之下)

当然我们也可以指定解释器执行

同时，就像 C 语言等等语言一样，我们都可以在命令行里面给 shell 传参。

变量

在 bash 脚本的撰写过程中不像 python 脚本那样建议用空格分隔一些东西，因为在 bash 脚本中空格是用来分割命令和不同的参数的

bash 变量的赋值，请注意，你赋的值都是字符串

变量的引用

由于空格以及元字符被赋予了意义，我们可以使用引号来强制的把引号（包括单引和双引号）之内的内容记录为字符串而不是让他有相应的意义。

我们想要在一个字符串中引用一个变量，那么我们需要使用双引号来保证能够替换；
而单引号则不能替换，会将内部所有成分都转化为字符
