发表于2022-07-09|更新于2022-07-26|Computer Science
|字数总计:951|阅读时长:2分钟|阅读量:

目标导向：

对于数据科学的处理，我们通常有一套较为长而且复杂的数据处理管道，其中涉及到多个文件之间十分复杂的依赖关系.

倘若每一次数据处理都需要手动的一层一层执行流水线的每一步，那么当数据有更新或者流水线过于复杂等等的时候，就不利于其他的工作者复现你的数据处理过程.

于是乎，就有了一种通用的自动的且规范化编译数据流水线的需求，那么GNU Make等等自动编译的工具应运而生.

什么是Makefile

makefile 就是编译的过程

makefile 用来处理大量的文件编译的问题，如果不按照规范处理大量文件编译的问题，很容易会出现编译错误.

makefile 就相当于告诉编译器如何编译，像一个Shell脚本一样，mf文件告诉编译器哪些文件先编译，哪些文件后编译，哪些文件重新编译等等.自动化的完成数据流水线

也就是说MF是一种类似于Verilog的描述性语言，描述文件之间的输入输出的编译关系

由于MF的可读性超级差，并且不能跨平台使用，因此我们现在多用CMake管理需要用MF来帮助编译的 C++ 项目（当需要编译的东西越来越多的时候）如果需要了解CMake，请跳过之后对MF语法的介绍

小的项目可以用一用

GNU make工具

make就是一个命令工具.

它用来解释MF文件中的指令，从MF文件中获得如何编译我的文件的信息，

我们只需要以MF的方式告诉GNU make文件之间生成的逻辑关系即可，而在一次编译过程中中间文件缺少没有，并不需要我们关心，Make工具会根据MF告诉的编译过程自动帮我们生成缺少或者是版本落后的中间文件，大大的自动化了我们处理大量的数据流

下面我将数理MF的语法

语法

一个MF文件包括一系列编译的规则，这是MF的基本语法单元

其中target是我们在这个规则希望生成的文件的名
而pre1，pre2…等是这个规则生成文件过程中需要输入文件
而command就描述的是生成过程，一系列命令

变量

引子
在一个MF的文件之中可能存在一些规律有很长很长的prerequisites.如果我们希望加入新的prerequisite，那么可能需要修改很多很多地方，那么如果有一个地方没看到，那就G了（笑

为了方便，MF引入了变量机制，在MF中，变量就是一个alias标签，用来代替某一个字符串，像C里面的宏定义，因此在使用的时候只需要将变量字符串填入使用宏定义的地方再对代码进行理解即可

在MF中，变量赋值有两种方法

  1. eager evaluation: :=
    立即扩展
  2. lazy evaluation: =
    延时扩展
    递归扩展
    (等有时间再解释，先用着eager evaluation)

当我们需要引用变量的时候，我们用的语法就是$(<var_name>)，用dollar符号直接引用
同时，MF内置了两个特殊的变量target和prerequisites表示当前规律的生成文件名和输入数据.当我们要引用的时候使用$^ (输入)和$@（输出）
