C/C++ Compiler Tools on Linux

Multi-File Compiler Tools

目标导向：

对于数据科学的处理，我们通常有一套较为长而且复杂的数据处理管道，其中涉及到多个文件之间十分复杂的依赖关系。

倘若每一次数据处理都需要手动的一层一层执行流水线的每一步，那么当数据有更新或者流水线过于复杂等等的时候，就不利于其他的工作者复现你的数据处理过程。

于是乎，就有了一种通用的自动的且规范化编译数据流水线的需求，那么 GNU Make 等等自动编译的工具应运而生。

什么是 Makefile

makefile 本质上就是将如何编译的过程以一种模式记录了下来

makefile 用来处理大量的文件编译的问题，如果不按照规范处理大量文件编译的问题，很容易会出现编译错误。

makefile 就相当于告诉编译器如何编译，像一个 Shell 脚本一样，mf 文件告诉编译器哪些文件先编译，哪些文件后编译，哪些文件重新编译等等。自动化的完成数据流水线

也就是说 MF 是一种类似于 Verilog 的描述性语言，描述文件之间的输入输出的编译关系

由于 MF 的可读性超级差 (语法不直观)，并且不能跨平台使用，因此我们现在多用 CMake 管理需要用 MF 来帮助编译的 C++ 项目（当需要编译的东西越来越多的时候）如果仅仅需要了解 CMake，请跳过之后对 MF 语法的介绍

P.S. 小的项目可以用一用 MF

GNU make 工具

make 就是一个命令。

它用来解释 MF 文件中的指令，从 MF 文件中获得如何编译我的文件的信息，

我们只需要以MF的方式告诉 GNU make 命令我的文件之间生成的逻辑关系即可，而在一次编译过程中中间文件缺少没有，并不需要我们关心，Make 工具会根据 MF 告诉的编译过程自动帮我们生成缺少或者是版本落后的中间文件，大大的自动化了我们处理大量的数据流

变量

引子
在一个 MF 的文件之中可能存在一些规则有很长很长的 prerequisites.如果我们希望加入新的 prerequisite，那么可能需要修改很多很多地方，那么如果有一个地方没看到，那就 G 了（笑

为了方便，MF 引入了变量机制，在 MF 中，变量就是一个 alias 标签，用来代替某一个字符串，像 C 里面的宏定义，因此在引用变量的时候只需要将变量字符串填入引用的地方再对代码进行理解即可
