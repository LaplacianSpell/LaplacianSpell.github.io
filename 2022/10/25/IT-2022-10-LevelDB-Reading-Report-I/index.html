IT-2022-10-LevelDB-Reading-Report-I

发表于2022-10-25|更新于2023-09-25|Computer Science
|字数总计:533|阅读时长:1分钟|阅读量:

本文为记录阅读 LevelDB 源代码的笔记系列第一作 (持续更新中)

此外还附带有一些如何阅读大型代码（非 ToyModel）的建议

本文先概览文件工程的结构

代码结构

总体上，LevelDB 的工程文件结构如下

  • ./assets 媒体文件
  • ./benchmarks 压力测试代码
  • ./include 头文件，存放对外接口
  • ./util 测试代码，单元测试(unit)，对最小的单元进行测试，比如函数，模块等等，一般通过 Github 上 CI 自动进行，保证 Merge 的安全
  • ./issues 测试代码，用于测试在 Github 上开 issue 发现的 bugs 修改过后的代码单元

对外接口（./include 下文件）

./include/abstract_db.h

声明一个虚接口 AbstractDB 在 ./include/abstract_db.h 中

  • abstract_db.h

    • struct Options/ReadOptions/WriteOptions... 提供给数据库相关方法的选择，即告诉数据库相关方法传入的类型是什么，从而调用相应的方法正确的实现

    • class Snapshot 数据快照，实现高并发读取数据

      • 数据结构: 环状双向链表

    • struct Range

    • class AbstractDB 接口的核心代码

      在抽象接口中，定义了对数据库的操作的基本方法( Open/Delete/Put/Write... )

内部代码(./db 下文件)

./db/db_core.cc

这一部分是实现 AbstractDB 的核心逻辑。

  • ./db/db_core.h

    • class DBCore : public AbstractDB 对所有虚接口的实现

./SkipList.h

文件 ./SkipList.h 这一部分是 DBCore 的内置数据结构实现——跳表（没有.cc文件是因为模板不能将声明与实现分开）

将这一数据结构的实现放在这里是为了将数据结构与接口调用分离.这样我们可以对数据库的接口写多种数据结构实现

数据库即键值对，希望实现快速的增删改查.

跳表 是基于有序二分查找 + 链表的思路形成的数据结构，有既好用又好写的优势，Redis 用跳表不用红黑树，简单来说就是通过附加一些指针，使得能够快速的跳到我要进行二分查找的 middle 位置
