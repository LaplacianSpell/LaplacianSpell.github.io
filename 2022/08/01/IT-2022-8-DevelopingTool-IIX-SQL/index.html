关系代数

我们知道，代数就是集合 + 公理 + 映射

作为数据库处理的相关数学原理基础，关系代数所基于的集合是集合的集合.

我们把一个数据库里面不同的表格看作不同的集合，而集合内部的元素就是表格的每一行，也称作关系，也就是说，关系代数描述的是不同集合之间的运算与约束运算的公理。

之所以我们将每一行称作关系，实际上是有很深的意义的。这暗示着每一行的元素之间是等价的，比如，假如某一行是这样的：

姓名 性别 ID
Li Male 01

按照常理理解，我们一般是以 ID 找信息条目。但是在表格里不管你的元素是什么意义，一行只代表一种关系
$$(Li,Male,01)$$
他们之间可以相互索引，在一行以内只是表明他们之间存在一个松散的联系，这也是为什么我们不把他称作条目什么的词汇的原因。

一个数据库的结构如下

  • filename.db
    • table A
      • relation 1
      • relation 2
        …
    • table B
      …

而我们定义在这个集合上的基本映射有

  1. 集合运算：∩,∪,\ （交，并，差）

  2. 线性运算：笛卡尔积，投影，选取。

严谨的数学告诉我们，只要在一个集合中实现了上述所有的运算，我们就能够按照需求取出我们想要的所有的数据。

SQL

表格的单位是关系，一个关系就是一行,而 SQL 就是驱动不同的表格（也就是关系的集合）之间的实现上述关系代数的六大运算的强力工具。

SQL（Structured Query Language，结构化查询语言），描述的就是如何利用六种基本操作实现相关的目的查询。

SQL 中大部分命令是大小写不敏感的。

第一个命令，创造表 A 并指定表的名称 A 以及内容的数据类型

第二，三个是插入操作。指定表格插入关系

最后一个是将表格 A 中所有的内容全部打印到标准输出

注意每个命令结束的时候以;作为标记
