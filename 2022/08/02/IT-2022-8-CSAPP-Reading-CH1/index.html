C 程序的生命周期

C 程序实际上以一个文本文件的方式存储在磁盘上，而他为了被系统读懂，最终需要转化为一系列机器语言。这些机器语言指令被打包成可执行目标文件（.exe）

C/C++程序生成一个可执行文件的过程可以分为4个步骤：预处理（Preprocessing）、编译（Compiling）、汇编（Assembly）和链接（Linking）。

  • 预处理：在编译器处理程序之前完成头文件的包含（预处理器会直接将上述头文件中的内容插入代码中），宏扩展（将#define后的东西在相应的地方直接插入），条件编译，行控制等操作。生成.i文件
  • 编译：通过词法分析和语法分析，在确认所有的指令都符合语法规则之后，将源文件代码翻译成等价的汇编代码。.s
  • 汇编：将汇编语言代码翻译成目标机器指令，生成目标文件。.o
  • 链接：将有关联的目标文件（以及库）相组合为一个可执行文件。.exe

而要执行上述全过程，我们需要使用编译工具，Linux 系统上的编译工具有 gcc 以及 g++

系统的硬件组成

实际上可以大致分成四大块，左上方框以内的封装为 CPU ，中间的系统总线，I/O 桥，存储器总线和 I/O 总线都封装为总线, 剩下的两个分别是 主存 和 I/O设备（下面的适配器 + 控制器）

  1. 其中总线就是计算机中的电子信息通路，传送定长的字节块，也就是字（word），字节块的字长是一个参数，在不同的系统中可以不一样。

  2. CPU 从不同的存储器读取数据的快慢是不一样的，

    从磁盘驱动上读取数据就比从主存（DRAM, 动态随机存取存储器）上读取数据开销大 1000w 倍，但是磁盘驱动的空间大 100 倍左右。

    类似的，从 registers 上读取数据比从 DRAM 上读取数据快 100 倍左右，但是寄存器上只有几百字节的空间，

    为了缓冲这种差距，还有一种称为高速缓存（cache）的部件，采用的是静态随机访问存储（SRAM）技术，其本身也分为 L1（芯片上的高速缓存）、L2（芯片外的高速缓存）L1 比 L2 快，比寄存器慢。L1,L2 都比主存快，作用是暂时集结信息，存储可能会被处理器用到的信息。

P.S.: SRAM 和 DRAM 技术的实现，可以参阅李国林老师撰写的《电子电路与系统基础》

操作系统的抽象

上一份关于操作系统的简单介绍, 针对一些基本概念的理解，过于浅，这里是更进一步的介绍。

操作系统有两个基本的功能

  1. 保护硬件，及时停止失控的程序。
  2. 向程序提供统一的接口调用各种硬件。

这就需要操作系统对硬件实现抽象.

操作系统对硬件实现了三个抽象：

  1. 抽象 I/O 设备为文件
  2. 抽象主存与磁盘文件为虚拟内存
  3. 抽象正在运行的程序为进程

其实光这么说什么信息都没有，你只要记得操作系统将我们对硬件的操作抽象的封装为调用操作系统的一些 API，然后这些 API 被集合成各种概念…

进程

进程是操作系统对正在运行程序的一种抽象。

操作系统上往往同时运行多个进程。

但是，单核 CPU 只能在一个时间片内运行一个程序，即使我们拥有多核 CPU，能够同时运行的程序数目也小于真正同时运行的进程数。

这是因为操作系统通过上下文切换机制，在一个系统上同时运行多个进程，即实现了对多个进程的并发运行

举例：单核 CPU 的上下文切换机制。

  1. Shell 进程阻塞在等待输入的位置，当我们输入执行程序的名称的时候，他会解析字符串。
  2. 当解析完毕，知道要调用的程序是什么之后，shell 进程会调用一个特殊的函数系统调用（syscall），函数会将控制权交给操作系统内核，内核会将当前进程的信息（比如主存内容，PC 和寄存器内容等等）保存下来，为了能够之后回到当前状态
  3. 之后，操作系统内核会开一个新的进程执行要求的程序并将控制权交给新进程
  4. 当上述进程结束之后，会将控制权交还给操作系统，操作系统会恢复原来进程的上下文。之后将控制权传递给原来的进程

上下文转换是由特殊的程序操作系统内核实现的，

此外，一个进程可以有多个线程.不同于进程之间被操作系统隔绝，同一进程的不同线程可以共享进程资源。

虚拟内存

虚拟内存，是将存储空间抽象成即使对应的物理位置不一样，但对每一个进程仍然保持相同结构的产物。

他为每一个进程提供了一个面对的存储器都一样的假象。
