IT-2023-2-CSAPP The Representation of Information in Computer Systems

发表于2023-02-21|更新于2023-09-25|Computer Science
|字数总计:3.1k|阅读时长:11分钟|阅读量:

The int $\ne \N$, the float $\ne \R$

How to represent and store information in computer?

The storage of information

  • 信息的表示：We use Binary to represent information in computer.

    • 我们需要将所表达的信息 编码 成为二进制数，比如 int 类型的 2-补码，float 的浮点数等等

  • 信息的储存：字节是计算机的最小可寻址的储存单位

    • 对于一个进程来说，虚拟内存就像一个 byte 数组
    • 每个 byte 的 address 是按照顺序从 0x00000 开始的
    • 1 byte = 8 bits, 而十六进制数的每一位都要用一个 4bits（$1111_2 = 15_{10}$）来进行编码，也就是说，一个十六进制数的每两位用一个 byte 进行编码

Word

  • Word 是计算机中 integer-value 数据以及 pointer 的标准大小，不同的计算机有所不同，实际上字长就是计算机中一个寄存器的大小。
    • 因此，整数值中的 long int 会随着机器不同而不同 32bits 机器是 4byte 而 64bits 是 8bytes.
    • pointer 类型的大小也有所不同（因为要顺序编码所有虚拟内存的地址）
  • 由于虚拟地址以一个 Word 来进行编码，因此一个字能够表示的所有可能编码的大小就是 虚拟内存 的最大空间大小，（32-bits with $2^{32} = 4GB$）

Byte Ordering

在大部分机器中，对于 多字节的对象 ，一般都是存储在连续地址的字节中的。这样对于一些特殊的数据类型就有不同的存储方式

Example: Store of int i = 0x 01 23 45 67

大端字节序：数的最高位放在地址的最低位，数的最低位放在地址的最高位

0x0000 0x0001 0x0002 0x0003
01 23 45 67

小端字节序：vice versa (地址与数字的高位一样)

0x0000 0x0001 0x0002 0x0003
67 45 23 01

这个问题虽然对于应用级别的程序员不关心（因为在相同的机器上运行的结果编译产生的情况都是一样的）但是：

  • 网络传输中极其重要，因为当我们在不同存储方法的机器间发送数据时，这样的存储方法会造成读取数据的错误
  • 检查机器级（非汇编）程序时，同样需要注意这个问题
  • 在强制类型转换时

也因为是只有多字节对象才会有大端序和小端序的担心，所以 char
类型等就不会有相应的担心

The Representation of interger

Now we begin to talk about the representation of interger.

The two representations: unsigned and int

对于无符号整数，我们采用的编码方式是 原码 (w 是 bits 数):

$$
B2U(x) = \sum_{i=0}^{w-1}x_i·2^i
$$

这种编码方式实际上就是简单的十进制和二进制的转换

对于整数，我们采取 2-补码 的方式进行编码

$$
B2T(x) = -x_{w-1}·2^{w-1} + \sum_{i=0}^{w-2}x_i·2^i
$$

  • 用人话说，就是把后面 w-1 位的数字转换成十进制，然后看最高位（符号位）是否是 1，如果是 1，则需要减掉（本质上是一 mod 操作）

  • 在补码中，最高位 $w-1$ 实际上是符号位，0 表示非负，1 表示负数

  • 补码中负数与对应的正数之间的关系 $-x =\ \sim x + 1$，即对正数 按位取反 之后加一

Number Range

w-bits

种类 最大值 最小值
U $2^{w}-1$(进位 -1 1111…11) $0$
T $2^{w-1}-1$(等比数列求和 01111…1) $-2^{w-1}$ (只有上述公式中的第一项 1000…0)
  • 两种编码方式都没有浪费，能够编码的数的数目是一样的
  • T 的最大值比最小值的绝对值多一个的原因是编码了 0

The change between int and unsigned int

当二者都是 w 位表示的时候，只需要将对应的 binary 形式按照不同的编码方式进行解读即可。

实际上相当于：

  • 在正数区域，二者相同

  • 在补码的负数区域，相当于 $mod \ 2^w$

    (e.g. $-8_{T} \rightarrow 1000_{(b)}\rightarrow 8U \equiv -8(mod \ 16)$)

The expansion of numbers

  • 对于 unsigned 的增加字节数，我们直接在前面添加 0 位即可

  • 对于 int 的添加字节数，正数添加 0，负数添加 1（即，将负数符号位缩回来 $-2^{w+k-1}+2^{w+k-2}=-2^{w+k-2} …$）

  • C 中自动的实现了这一过程

truncating of numbers

高位直接丢掉，剩下的按照规则直接翻译

对于无符号数，截断为一个 k 位数字相当于 $x \ mod\ 2^k$

对于有符号数，我们先将其转换为无符号数，对无符号数按照上面取 mod 之后再转换为有符号数，实际上也就相当于对字节进行截断之后再按照补码进行翻译

Basic Operations of integer

Added

  • 对于无符号整数

    • 基本分析：对于 w 位的相加计算，我们最大会得到 $2^w - 1 + 2^w - 1 = 2^{w+1} - 2 < 2^{w+1}-1$ 这么大的整数。因此会造成溢出
    • 处理方法：我们直接将计算得到的 bits 进行 truncate 即可，相当于 mod 一个 $2^{w}$
    • 判断溢出：判断是否发生了溢出，我们可以比较 结果 是否比 加数两个中的任何一个小 即可

  • 对于有符号的 2-补码 编码数字

    • 基本分析：对于 w 位（$-2^{-w} < x < 2^{w-1} - 1$）的加法运算，我们得到的范围为 $-2^{-(w+1)} < x < 2^{w} - 2$，需要 w + 1 位
    • 处理方法：我们也是 bits 相加之后 truncate，按照有符号的方式进行读取
    • 判断溢出：此时会有溢出的结果
      • 当两个正数相加得到一个负数，即 01… + 01… => 1… < 0
      • 或者是两个负数相加得到了一个正数，即 1… + 1… => 1 0… 然后最高位被剪掉了
      • 也就是说，结果符号不同 或者是最高位和符号位的进位是否相同，如果只进位一个，比如只进位到符号位或者是符号位进位到超出 w 宽度，则表示发生了 overflow

Multiply

计算一个精确值，之后按照位数截断。 $w$ 位计算之后需要 $2^w$ 位表示结果

移位

左移操作相当于 $u * 2^k$ ，即在原有的数字上乘 2，当超出范围时截断

右移操作相当于 $u / 2^k$ 之后向下取整（直接理解成抹掉小数点）

The representation of float

按道理来说，计算机里面确实是可以表示小数点的，我们有 2 进制分数，例如 $1.01_2 = 1 + 1/4$ 和十进制中小数点后面的每一位是 $1/10^n$ 道理一样。$0.111111… \rightarrow 1$ 可以 notate as $1-\varepsilon$

但是直接按照位数编码会导致一些问题，比如不能表示某些循环小数，以及太大和太小的数（需要 0.00000…很多位才能看到有效位）等（因为这种表示方法对于小数部分是不均匀的）

例如 1/3 = 0.0101010101010…

IEEE 754

在 伟大的 IEEE 754 标准登场之前，各大厂商的计算机遵循不同的浮点记录方式，这为程序复用造成了极大的障碍，导致浮点运算变成了计算机科学家 pong 都不能 pong 的滑梯（credit: double-win）

这个标准巧妙的采用了对 $(-1)^s \times M \times 2^{E}$ 的$s,M,E$ 进行编码而不是对数本身进行编码的方式，实现了浮点数。

  • $s$: 符号位 0 表示正数，1 表示负数，对于浮点值为 0 的情况特殊考虑。
  • $M$：一个 二进制小数，范围在 $0 \sim 1-\epsilon$ 之间或者是$1 \sim 2-\epsilon$ 之间
  • $E$：是 2 的幂，可能是负数，用来移动$M$小数点

对于单精度浮点格式（4bytes 32bits）：

s exp frac
1bits 8bits 23bits

对于多精度浮点格式（8bytes 64bits）：

s exp frac
1bits 11bits 52bits
  • Normalized Number：对于指数区域 exp 不是全 0 或者全 1 的情况（单精度为 0/255，双精度为 0/2047）
    • 在这种情况下，读取 $E = \exp - bias$
      • exp 为 exp 区域编码的无符号数字
      • bias 为一个 $2^{k-1} - 1$ （其中 $k$ 为 exp 占位的宽数）
      • 也就是导致最终指数区域编码的 $E$ 的取值范围是 $-2^{k-1}+1\sim 2^{k-1} = 2^{k}-1 - (2^{k-1}-1)$
    • 同时对 frac，读取为 $M = 1.frac$ 用 frac 编码 M 的小数位。同时 implied leading 1，巧妙地添加了一个精度。
  • ~Normalized Number：
    • 当指数区域全零，指数值为 $E=1-bias$（不是 0-bias 是为了使得二者之间平滑过渡）而有效数字为 $M = 0.frac$ .非规格化数表示 0：
      • frac = 000…,exp = 000…,s = 0，表示 +0.0
      • frac = 000…,exp = 000…,s = 1，表示 -0.0
    • 当指数区域全是 1，小数区域全是 0 时，非规格化数字表示 $\infty$：
      • 当 s = 0 时候表示 $+\infty$
      • 当 s = 1 时候表示 $-\infty$
    • 当指数区域全是 1，小数区域不是 0 时候，表示 NaN, 当出现非法运算时会返回这个结果

Example

  1. 当 exp = 000…，时候，如果 frac 也是 0，即 0.000…,我们理解成 0.0；如果 frac 不是 0，我们理解成移小数点向左很多位的很小的一个数字
  2. 当 exp = 111…, 时候，如果 frac 全是 0，则表示最大值（inf），再大一点，比如 frac 不全是 0，则不是数
s exp frac
1 4 3

bias = $2^{4-1}-1 = 3$, 所以移位范围是$-3\sim4$

Addition/Multiplication and Rounding of float

对于浮点计算，实现的方式是两步

  1. 计算精确结果。
  2. 按照 舍入规则 进行舍入。

那么，舍入规则就变得十分的重要，在 IEEE 754 中 舍入规则是所谓的 rounding to the nearest

  • 首先是向最近的有效数舍入
  • 如果它与两个相邻的有效数距离一样时（即它是中间数，halfway），那么舍入到最近的偶数有效数。

当我们见到一个二进制表示的分数的时候，其实舍入规则可以转变为：

  • 有效位的后一位是 0，则表示当前数距离不舍入的那个数更加靠近一些，不舍入。
  • 有效位的后一位是 1 且之后的位不全为 0，则表示当前数距离舍入后的那个数靠近一些，需要舍入。
  • 特殊的，如果有效位后一位是 1 且之后全部是 0，则需要看距离最近的偶数是谁，舍入到上面去

浮点运算不满足 加法的 Abelian Group，因为一旦涉及到溢出，结合律可能不满足

float in C

C 中的浮点数可以与 int 等相互转化。

  1. float 类型转换到 double 类型，可能会溢出到 无穷，同时由于精度发生改变，可能会有 舍入
  2. int 转移到 float 不会溢出，但是可能会舍入，因此当转移到float 又 转移回来之后可能与原来值不相等
  3. 浮点数转换为整型，会向 整数 直接截断，即把分数部分直接扔掉，如果发生溢出，则取为整型能表示的最大或者最小值
