Simple Introduction to Process on GNU/Linux

Process

We often encounter several different concepts: the process, the thread, and the program. the program is a static concept to describe a kind of special files in our computer, while the process is a dynamic concept. It describe a progress of our computer to run a program.

Our computers are able to run several different process at the same time, i.e. the “cocurrence”. in which the computer resources are able to be shared between different process.

For the management of Processes, the OS use PCB(Process Control Block) to store and descibe the basic infomation of each process. In GNU/Linux, the PCB is implemented as struct task_struct.

The Signal

The signal is a special asynchronize message sent to process. When the process receives a signal, it will handle the signal immediately.

In GNU/Linux, the signals are specified with different numbers, and we use these signals by there name in a program. These names are macros (e.g. #define SYGINT 2).

In glibc, we have struct sigaction in signal.h

and we can set a signal with sigaction by int sigaction( int signum, const struct sigaction * act, struct sigaction * oldact )

the signum is the unique signal number, and act is the new signal action we want the signal to have. oldact is the old signal setting, usually set to NULL.

Creat a Process

  1. We can use system(const char * cmd) to create a child process. on which the command is executed.the function will return the state code.

  2. We can use fork() to fork the current father process to generate a child process,after a new child process is created, both processes will execute the next instruction following the fork() system call. the function will return 0 in child process, while return the PID of the child process in father process. So we can execute different operations in two processes.

Terminate the Process

  1. We can use the int kill( pid_t pid, int sig ); to kill the given process with its pid, the sig can be set as SIGTERM, which is the termination signal.

  2. We can use wait(int* status) to suspend the father process to wait for child process and resycle the PCB.

    一个进程在终止的时候会关闭所有的文件描述符，释放在用户空间分配的内存，但是它的 PID 还保留着，内核在其中保存了一些信息：如果进程是正常终止则保存进程退出状态；如果进程是异常终止，则保存导致该进程终止的那个信号。这个进程的父进程可以调用 wait() 或者 waitpid() 来获取这些信息，然后彻底清除这个进程。 (see ref[1])

    We can use WIFEXITED(status) to justify whether the child process exits normally.if so, Then the status code can be read with marco WEXITSTATUS(status).

if we DO NOT use the wait() to recycle the PCB of child process.There are 2 possible cases:

  1. Father process exits before the child process exits. And then child process becomes an orphan process and adopted by init process(PID: 1), then it can exit normally under the management of init.We can also use kill to kill the orphan process.
  2. Child process exits before the father process exits. And then the child process becomes the zombie process. We can only kill the father process to let the init to recycle the zombie process.

Communication between Processes

Pipes

Pipes are half-duplex devices(remember what I’ve metioned in I, devices are file in GNU/Linux) that are only used to communicate between 2 processes with “affinity” (e.g. father -> son).

The data are serially transported and read.

  1. Create a Pipes: We can use int pipe( int pipefd[2] ); to create a pipe. The function will return the status code. what is really important is pipefd[2].After we call the func, they will become 2 file descriptors, the “0” one is READ FILE, the “1” one is WRITE FILE.

  2. Redirect the File Descriptor: We can use int dup(int oldfd); to return a new file descriptor and bind it to the same file which has already been pointed by int oldfd.

    When we want to specify the new file descriptor, we can use int dup2(int oldfd, int newfd);.

    Remember to use close(newfd); before binding it to the new file.

Semaphone

The GNU/Linux has implemented 2 types of semaphores:

  • System V semaphores (sys/types.h, sys/ipc.h, sys/sem.h), Usually used in sychonization of Process
  • POSIX semaphores (semaphore.h…), Usually used in Threads sychonization
