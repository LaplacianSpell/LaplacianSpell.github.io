The first thing I wanna to elucidate is how to handle the command line in C++.

getopt.h in GNU/Linux to parse the command line

getopt is a C library that provides some useful functions for parsing command.

there are two important stuffs.the first thing is struct option, which is a structure with 4 elements:

  1. const char* name: the long name of the argument. e.g. -h -> –help
  2. int has_arg: a flag to show whether the option have an argument. e.g. 0 -> none, 1 -> yes, 2 -> optional.
  3. int * flag: use to store the val, set as NULL
  4. int val: store the short name of the argument in the form of ASCII.

the second thing is int getopt_long( int argc, char * const * argv, const char * short_options, const struct option * long_options, int * long_index );

  1. we give the argc and argv arguments of main() to the function.

  2. the short_options argument, pointing to a string containing the all short options.e.g. -h,-o <arugumet>, -p …. we fill all the options we want to add to this command. such as short_options = "ho:p" which means we have h, o, p three options and o can follow some arguments.

  3. the long_options is a array of structs filled with struct options.

the last are set to NULL

they are all defined in the glibc.

How to handle the Environment Variables in GNU/Linux

The methods are same. We use getenv(char* variable_name) in cstdlib.h

And we also have a variable to store the environment variables char** environ, which is also defined in glibc, this is a 2-D array with nullptr as the last element to stop the read circling.

How to handle the Syscall Error?

When we conduct a syscall operation, and the system returns an error. We can use int erron in erron.h to get the latest errorcode (the error code will change after each syscall, so we must store it and then do other processing by using the stored errorcode other than straightly using erron)

the standard error code are uppercase-character macros starting with E. e.g. EPERM which means the operation is denied by OS kernel.

we can use strerror(<erro_code>) in cstring.h to get the additional information on the error code.

The management of source in GNU/Linux

There are sort of different source in GNU/Linux. And we should handle them properly.Usual steps are as follows:

  1. portion the source.
  2. conduct the process on utilizing the source.
  3. if the process fail, then we should release the source and exit normaly.

The syslog info and user info

the system using 2 deamon processes to maintain the logging information.

Stream, File, and devices

I/O streams

we have two streams

  • std streams: cin, cout

    these streams have buffers so only if the buffers are flushed or shut down, the data will be written.

  • std err: no buffer

file descriptors

In GNU/Linux, when we open a file, Kernel will maintain a list of file descriptors for us to uniquely identify the file.

the file descriptors are a int type element which is a positive integer.

since every thing in the GNU/Linux is files, the Kernel abstract all these things into a general interface in the form of file descriptors.

There are also C standard stuffs and C++ streams for us to read and write files, but GNU offers file descriptors.

I’ve omitted the demos for kinds of functions to manipulating files in GNU/Linux ,Since I’m kind of lazy.But you can always find them in official documentations by yourself.

temp file

If we want to use a temporary file, we can create it with mkstemp(cha* filename) function and then we’ll have a temporary file named with random 6 characters in /tmp/.
