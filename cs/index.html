  • 运行时（RunTime）
    三个含义

    指「程序运行的时候」，即程序生命周期中的一个阶段。例句：「Rust 比 C 更容易将错误发现在编译时而非运行时。」
    指「运行时库」，即 glibc 这类原生语言的标准库。例句：「C 程序的 malloc 函数实现需要由运行时提供。」
    指「运行时系统」，即某门语言的宿主环境。例句：「Node.js 是一个 JavaScript 的运行时。」

  1. X-server: X server 是 Linux 系统里面图形接口服务器的简称。Windows 系统的界面是这个系统不可分割的一部分，各种窗口操作界面显示都是由系统核心直接管理的，而 Linux 的图形界面并不是系统的必要组成部分，它可以在无界面的条件下运行。当需要 Linux 提供界面的时候，系统就会建立一个或者数个 X server，通过 X 协议跟窗口管理器交互，由独立于系统的应用程序来产生窗口，状态栏，按钮之类的交互界面。

比较常见的 Linux 界面操作环境有 KDE 和 GNOME，为它们提供系统支持的就是 X server，而并非 Linux 核心。总结一下 linux 图形界面层次关系：

linux 本身–>X 服务器<-[通过 X 协议交谈]->窗口管理器（综合桌面环境）–>X 应用程序。

  1. vim 打开时间慢的问题

    今天发现不知道为什么 vim 打开所需要的时间巨nm长（差不多需要1min多），难以忍受，于是上网查阅相关的资料后发现了很多人和我有一样的问题(issues5223),不过似乎涉及到了许多我不知道的例如 GNORME, Xserver 等等知识，需要进一步的学习.

    今天（2023/2/23）好像误打误撞的解决了上述问题，说实话也是在上述的 issues5223 启发之下工作成功的，原文中的一小段挂在这里

If the variable is set and an X11 server is running on Windows and reachable, then vim starts correctly.
If the variable is set and the X11 server is not reachable, then it times out after 30sec and the startup is therefore slow.
If the variable is not set, vim starts correctly.

于是意识到自己似乎没有打开 `X server` 导致 `vim` 打开的时候巨慢。
打开 `XLaunch`，问题得到解决。看来真的是图形界面的问题，不过原理自己还没弄明白。为什么 vim 需要使用 Xserver？？

  1. 今天删了一些插件，大概是之前不懂的时候乱装的，然后说实话自己可能也不理解然后就装了的插件，结果发现我的vscode不工作了，或者说 code 命令不工作了，翻来覆去找了各种信息，最后在官方文档上解决了我的问题，看来是不小心删掉了一些 remote 相关插件。从这里甚至了解到我们用 vscode 并不是真的在 WSL 上面装了一个 vscode， 而是在使用本地的 vscode 挂 WSL 的远程，之前自己实在是一直在瞎用…对 WSL 工作的方式越发的感兴趣了（）。

  2. 今天装了一个比较好看的 shell: zsh，同时添加了 oh-my-zsh，然后选了一个好看的 Theme 叫做 agnoster，结果发现有乱码，查了一下发现是 zsh 对于 Powerline 有依赖，需要预装相关的字体Noto Mono for Powerline，具体教程在这里，开始以为是装到 WSLinux 上，看来还是有点 Naive，我出现乱码完全是因为自己使用 Windows 上没有相关的字体而已，因此是需要在Windows 上装相关的东西，好了，现在把 git for windows 删掉了看来只能下载压缩包，下载下来发现 ./install.ps1 不能跑！报错没有对相关文件进行数字签名，查了一下似乎是 Powershell 上对脚本能不能跑有权限的限制之类的东西，叫做执行策略（详见巨硬相关文档），于是就在 Administrator 下先运行 Set-ExecutionPolicy -ExecutionPolicy UNRESTRICTED 人工不限制跑脚本的权限，下载安装完之后再手动调回 Default 即可，可以用 get-ExecutionPolicy 看当前的执行策略。我的 default 似乎是 Restricted. 不得不说感觉 ps 的逻辑有点笨重

    安装好之后在Windows Terminal 里面对我的分发 Profiles 采用对应的字体就好了。之后便有了那个三角形的 statuesline icon.

过滤 ASCII 字符

  1. abc 是 letter & 123 是数字
  2. \d 用来匹配一个数字，\D 用来匹配一个不是数字的
  3. . 用来匹配任何一个，. 用来转义，匹配 dot
  4. 上一个太强了，在给定范围内匹配一个，[abc]（匹配abc中的一个）
  5. 上一个有时候不太好用，在给定范围之外匹配一个，[^abc]（匹配 abc 之外的一个）
  6. 如果我们想匹配一个从a-h的所有字母，难道要全部列出来吗？不是，可以用 [a-h],对于排除也是同理的，同时还有一个只匹配正经的字符的 \w == [A-Za-z0-9]
  7. 重复匹配相同的模式，上面说的都是一个模式，a{3}匹配a三次，a{1,3}匹配 a 一到三次，不多于3不少于1，也可以混着使用[a-c]{1,6}
  8. Kleen star & Kleen plus: 有点像自由群的概念，将字符串进行自由组合，例如：a+(匹配至少一个a)，a*匹配0-infty 个a
